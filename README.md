# Term Life Insurance Pricing & Reserving Model

##  Project Overview
This project demonstrates an end-to-end actuarial pricing and reserving cycle for a 20-year term life insurance product. By decoupling the computational engine from the presentation layer, this project mimics standard industry practices for model governance and dynamic scenario testing.

##  Actuarial Concepts Applied
* **The Equivalence Principle:** Pricing baseline premiums such that the Expected Present Value (EPV) of Benefits equals the EPV of Premiums.
* **Mortality Assumptions:** Utilized standard life tables (e.g., SOA VBT) to extract $q_x$ rates and calculate cumulative survival probabilities (${}_{t}p_{x}$).
* **Prospective Reserving:** Calculated end-of-year terminal reserves by projecting future benefit outflows against future premium inflows.
* **Market Risk Assessment:** Modeled the inverse relationship between interest rates and premium requirements to quantify macroeconomic risk.

##  Tech Stack & Architecture
* **Python (pandas & numpy):** Acted as the backend computational engine. Prioritized vectorized operations over loops to efficiently calculate discount factors, EPVs, and the baseline reserve curve. 
* **Excel:** Acted as the locked-down frontend presentation layer. Built a clean, user-friendly dashboard utilizing data validation, named ranges, and dynamic scalar formulas (e.g., `XLOOKUP`) to prevent user error and protect the underlying data feeds.

##  Key Features
* **Dynamic Premium Scaling:** Users can input any custom Face Amount, and the dashboard engine instantly scales the baseline premium without needing to rerun the Python script.
* **Reserve Curve Visualization:** Plots the terminal reserve accumulation and depletion over the 20-year term, providing visual validation that the reserve successfully hits zero at maturity.
* **Interest Rate Sensitivity:** Features an automated "What-If" analysis, demonstrating how the required Net Level Premium shifts across varying interest rate environments (2% to 6%).

##  Repository Structure
* `Python_Engine.ipynb` *(or .py)*: The core script that cleans the raw mortality data, applies the financial mathematics, and generates the output tables.
* `master_actuarial_model.csv`: The baseline dataset generated by Python, acting as the immutable data feed for the Excel engine.
* `premium_sensitivity.csv`: The output of the Python loop testing various interest rate environments.
* `Term Insurance Pricing.xlsx`: The final interactive deliverable. 

##  How to Use
1. Download the `Actuarial_Pricing_Dashboard.xlsx` file.
2. Open the **Dashboard (Blue)** tab.
3. Adjust the **Face Amount** and **Target Policy Year** in the yellow input cells.
4. Watch the Key Outputs, the Sensitivity Analysis, and the Reserve Curve chart instantly update to reflect your new assumptions!
